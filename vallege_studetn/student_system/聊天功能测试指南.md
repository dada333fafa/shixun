## 🚀 测试步骤

### 1. 启动服务器

后端服务器已经在运行：
- 后端地址: http://localhost:5001
- 前端地址: http://127.0.0.1:8080

### 2. 登录学生账号

1. 打开浏览器访问: http://127.0.0.1:8080/login.html
2. 使用学生账号登录:
   - 用户名: `student1`
   - 密码: `student123`

### 3. 进入聊天页面

登录成功后，点击左侧菜单的"聊天沟通"或直接访问:
http://127.0.0.1:8080/student-chat.html

### 4. 查看教师列表

在聊天页面的左侧边栏，你应该能看到所有5位教师的列表：
- 王老师 (数学)
- 李老师 (语文)
- 张老师 (英语)
- 刘老师 (物理)
- 陈老师 (化学)

### 5. 开始聊天

1. **点击任意一位教师**（例如：王老师）
2. 右侧会显示聊天窗口
3. 在输入框中输入消息，例如："老师好，我想请教一个数学问题"
4. 点击"发送"按钮或按Enter键

### 6. 验证消息保存

**关键测试步骤：**

1. 发送几条消息给王老师
2. **切换到其他页面**（例如点击"仪表盘"）
3. **再回到聊天页面**
4. 再次点击王老师
5. ✅ **你应该能看到之前发送的所有消息！**

### 7. 与多位教师聊天

1. 点击不同的教师进行聊天
2. 每位教师的聊天记录都是独立的
3. 切换回之前的教师，消息应该还在

## ✨ 功能特点

✅ **消息持久化**: 所有消息都保存在MongoDB数据库中  
✅ **历史记录**: 刷新页面或切换页面后，消息不会丢失  
✅ **多对话支持**: 可以与多位教师同时聊天  
✅ **实时列表**: 左侧显示所有可聊天的教师  
✅ **时间显示**: 智能显示消息时间  

## 🔧 如果遇到问题

### 问题1: 看不到教师列表
- 检查是否已登录
- 打开浏览器控制台(F12)查看是否有错误
- 确认后端服务器正在运行

### 问题2: 发送消息失败
- 检查网络连接
- 查看浏览器控制台的错误信息
- 确认后端服务器的messages路由已加载

### 问题3: 消息没有保存
- 检查浏览器控制台是否有API调用错误
- 确认MongoDB数据库连接正常
- 查看后端服务器日志

## 📝 API端点说明

聊天功能使用的API端点：

1. **获取教师列表**: `GET /api/users/teachers`
2. **发送消息**: `POST /api/messages`
3. **获取消息历史**: `GET /api/messages/:userId`
4. **获取对话列表**: `GET /api/messages/conversations/list`
