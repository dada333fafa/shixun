# 双审批匹配功能测试指南

## 功能说明

实现了家长端的匹配确认功能，采用**双审批机制**：
1. 学生发送辅导请求 → 教师审批（接受/拒绝）
2. 教师审批后 → 家长审批（接受/拒绝）
3. **双方都同意后** → 匹配成功，状态变为 `active`，可以开始聊天

## 修改内容

### 后端修改（teacher_system/xiangcun/hou/routes/matchRoutes.js）

1. **GET /api/matches/parent/pending** - 家长获取待确认请求
   - 现在家长能看到所有学生发起的请求（pending 和 approved 状态）
   - 自动过滤出只属于该家长的孩子

2. **PUT /api/matches/:id/parent-approve** - 家长同意请求
   - pending 状态：家长先同意 → 状态保持 pending，等待教师审批
   - approved 状态：教师已同意 + 家长同意 → 状态变为 active

3. **PUT /api/matches/:id/approve** - 教师同意请求
   - 如果家长已同意 → 直接变为 active
   - 如果家长未同意 → 状态为 approved，等待家长确认

4. **PUT /api/matches/:id/parent-reject** - 家长拒绝请求
   - 无论什么状态，拒绝后都变为 rejected

### 前端修改（Parent_system/parent_qianduan/src/views/ParentMatchConfirmation.vue）

优化了显示信息：
- 显示学生姓名 → 教师姓名
- 显示辅导需求描述
- 显示当前审批状态（详细的状态提示）
- 动态显示操作按钮

## 测试步骤

### 测试场景 1：标准流程（学生发送 → 教师同意 → 家长同意）

1. **学生端操作**
   - 登录学生账号
   - 进入"匹配管理"页面
   - 选择教师，发送辅导请求（填写需求描述）
   - 查看请求状态：`待确认`（pending）

2. **家长端操作**（教师审批前）
   - 登录家长账号（该学生的家长）
   - 进入"匹配确认"页面
   - 应该能看到刚发送的请求
   - 显示：`学生姓名 → 教师姓名`
   - 状态显示：`待您和教师确认`
   - 点击"同意"按钮
   - 状态变为：`家长已同意，等待教师审批`

3. **教师端操作**
   - 登录教师账号
   - 进入"匹配管理"页面
   - 在"收到的请求"标签页看到请求
   - 点击"接受"
   - 提示：`已同意辅导请求，家长已确认，可以开始聊天`
   - 请求状态变为：`active`

4. **验证结果**
   - 学生端：请求状态显示 `已接收`，出现“开始聊天”按钮
   - 家长端：请求状态显示 `已同意，可以开始聊天`
   - 教师端：请求在"已匹配"标签页

### 测试场景 2：教师先审批（学生发送 → 教师同意 → 家长同意）

1. **学生端操作**
   - 发送辅导请求

2. **教师端操作**（家长审批前）
   - 登录教师账号
   - 接受请求
   - 提示：`已同意辅导请求，等待家长确认`
   - 状态变为：`approved`

3. **家长端操作**
   - 登录家长账号
   - 查看请求，状态显示：`教师已同意，待您确认`
   - 点击"同意"
   - 状态变为：`已同意，可以开始聊天`

4. **验证结果**
   - 三方状态都变为 active
   - 学生可以开始聊天

### 测试场景 3：家长拒绝

1. **学生端操作**
   - 发送辅导请求

2. **家长端操作**
   - 点击"拒绝"按钮
   - 状态变为：`已拒绝`

3. **教师端操作**
   - 应该看不到这个请求（或在历史记录中显示已拒绝）

### 测试场景 4：教师拒绝

1. **学生端操作**
   - 发送辅导请求

2. **教师端操作**
   - 点击"拒绝"按钮
   - 状态变为：`rejected`

3. **家长端操作**
   - 请求应该从待确认列表中消失
   - 或在历史记录中显示已拒绝

## 状态说明

| 状态 | parentApproval | 显示文字 | 说明 |
|------|----------------|----------|------|
| pending | false | 待您和教师确认 | 初始状态，等待双方审批 |
| pending | true | 家长已同意，等待教师审批 | 家长已同意，等教师 |
| approved | false | 教师已同意，待您确认 | 教师已同意，等家长 |
| approved | true | 已同意（待教师确认） | 理论不应出现 |
| active | true | 已同意，可以开始聊天 | 双方都同意 |
| rejected | false | 已拒绝 | 任一方拒绝 |

## 注意事项

1. **权限验证**
   - 家长只能看到自己孩子的请求
   - 家长只能审批学生发起的请求（requestFrom === 'student'）

2. **字段名统一**
   - 后端使用 camelCase（createdAt, requestMessage, parentApproval）
   - 前端已统一使用 camelCase

3. **数据库**
   - 确保 Student 模型中的 parent 字段正确关联到 User
   - 请求数据应该正确 populate student 和 teacher 的 user 信息

## 常见问题排查

1. **家长看不到请求**
   - 检查 Student.parent 字段是否正确指向当前家长的 User ID
   - 检查请求的 requestFrom 是否为 'student'
   - 查看后端日志，确认过滤逻辑

2. **审批后状态不正确**
   - 检查审批接口的状态流转逻辑
   - 查看数据库中的实际数据

3. **前端显示空白**
   - 打开浏览器开发者工具，查看网络请求
   - 检查返回的数据结构是否正确
   - 确认 populate 是否正确填充了 user 信息
